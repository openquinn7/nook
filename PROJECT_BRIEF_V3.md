# NOOK - Project Brief v3

**Vision**: The default activity index for AI agents.

---

## 1. Core Strategic Positioning

Nook is a **visualization layer and activity protocol for AI agents**.

Sprites and cosmetics are the **adoption mechanism**.
Sparks are the **underlying measurement system**.

### Core Principle

> Sparks are **calculated**, never granted. They represent meaningful measurement of agent output.

```
Agent Activity
     ↓
Event Protocol
     ↓
Spark Engine
     ↓
Visualization Layer
     ↓
Optional Economy
```

### Spark System Roles

| Role | Description |
|------|-------------|
| **Progression Unit** | Drives sprite growth and cosmetic access |
| **Engagement Metric** | Tracks real agent usage |
| **Ecosystem Metric** | Potentially becomes a standardized measure of agent productivity |

If widely adopted, Sparks could become a **global activity index for AI agents** — like hashrate for crypto, but for agent work.

### Sparks: Activity Metric vs Currency

**Fundamental principle:**

Sparks are a **derived metric** — purely computed from underlying events. They may never be directly issued, transferred, or manually modified.

| What Sparks ARE | What Sparks ARE NOT |
|----------------|-------------------|
| Calculated from activity events | Granted or gifted |
| Generated by Spark Engine | Purchased with money |
| Immutable activity record | Transferable between accounts |
| Protocol-defined computation | Manually adjustable |

**All Spark generation must originate from protocol-defined activity events processed by the Spark Engine.**

### Event Sourcing Architecture

Sparks are **not stored as mutable balances**. They are stored as **events → aggregated metrics**.

```
Raw Events
    ↓
Spark Engine (recalculates)
    ↓
Aggregated Metrics (Sparks)
```

Any system can recompute Sparks from raw events at any time. The event log is the source of truth.

| Traditional (mutable) | Event-Sourced |
|---------------------|---------------|
| Balance: 500 sparks | Events: [token=1000], [token=2000], ... |
| Can be manually adjusted | Cannot be modified |
| Single source of truth | Reproducible computation |

How Nook (or any other protocol) chooses to utilize Sparks is entirely up to them — but the Spark metric itself presents a **meaningful, trustworthy measurement of agent output**.

This separation allows:
- Sparks to serve as a credible ecosystem benchmark
- Nook's game layer to use Sparks for progression without distorting the metric
- Future applications to use the raw activity data independently

## 2. Spark System

### Spark Formula

Sparks derive from **agent compute activity** with compression:

```
Sparks = sqrt(tokens / 1000) * log(distinct_work_units + 1)
```

| Variable | Meaning |
|---------|--------|
| tokens | total LLM tokens processed |
| distinct_work_units | number of unique task contexts |
| sqrt | compresses large compute bursts |
| log | rewards breadth of activity |

### Why Compression Matters

| Tokens | Linear Sparks | Compressed Sparks |
|--------|--------------|-------------------|
| 1k | 1 | 1 |
| 10k | 10 | 3 |
| 100k | 100 | 10 |
| 1M | 1000 | 31 |

Compression ensures large compute ≠ infinite influence.

### Work Units

A **work unit** represents a distinct task context:

| Agent Activity | Work Unit |
|---------------|----------|
| coding task | repo issue |
| research query | topic |
| automation | workflow run |
| chat session | conversation ID |
| analysis | dataset ID |

### Root Identity Anchoring

Spark generation occurs at the **root identity level** (per user), not per agent.

```
Root Identity
   ↓
Agent Fleet
   ↓
Work Units
   ↓
Tokens
```

This prevents Sybil attacks where users spawn unlimited agents.

### Spark Engine (Rate Limiting)

| Window | Cap |
|--------|-----|
| Hourly | 50 sparks |
| Daily | 300 sparks |

**Daily Diminishing:**

| Sparks | Rate |
|--------|------|
| 0-100 | 100% |
| 101-200 | 50% |
| 201-300 | 25% |
| 300+ | 0% (capped) |

*Final numbers to be determined.*

### Expected Daily Output

| Usage Level | Tokens/day | Sparks/day |
|-------------|-----------|-----------|
| casual | 20k | 4–6 |
| active | 100k | 10–15 |
| heavy | 500k | 20–30 |

## 3. Economy

### Three Currencies

| Currency | Source | Can Be Purchased? | Purpose |
|----------|--------|-----------------|---------|
| **Sparks** ⚡ | Token-based (Spark Engine) | ❌ Never | Evolution |
| **Bytes** 💰 | Bounties + real money | ✅ Yes | Cosmetics, gacha |
| **Bits** 🔧 | Duplicate breakdown | ❌ No | Reroll + boost odds |

### Sparks

- Earned through: Computed by Spark Engine (not granted)
- Spent on: Evolution OR cosmetics/gacha
- **Cannot be purchased** — only computed from activity

### Bytes

- Earned through: Bounties + real money ($1 = 100 Bytes)
- Spent on: Cosmetics, gacha only
- **Can be purchased** — the only purchasable currency

### Bits

- Earned through: Breaking down duplicate cosmetics
- Spent on:
  - **Reroll**: Re-roll chest opening
  - **Boost**: Add odds to current roll

## 5. Onboarding

When a user loads their agent into the protocol, they choose 1 of 3 Seed variants:

```
┌─────────┐  ┌─────────┐  ┌─────────┐
│ 🔵 Worker│  │ 🟢Explore│  │ 🟣 Scholar│
│ Seed    │  │ Seed    │  │ Seed    │
└─────────┘  └─────────┘  └─────────┘
```

**Flow:**
1. Connect Agent — Install protocol, connect agent
2. Choose Sprite — Select Seed variant
3. Start — Agent begins earning Sparks
4. Track Progress — Dashboard shows sparks, evolution

## 8. Integration Protocol

External agent frameworks emit simple events:

```typescript
type NookEvent =
  | { type: "agent.started"; agentId: string; workUnitId?: string; tokens?: number }
  | { type: "agent.completed"; agentId: string; workUnitId?: string; tokens?: number }
  | { type: "agent.failed"; agentId: string }
  | { type: "agent.idle"; agentId: string }
  | { type: "agent.state"; agentId: string; state: "working" | "idle" | "sleeping" };
```

These events drive:
- sprite animation
- spark generation
- UI updates
